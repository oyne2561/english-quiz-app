# プログラミング英単語集

## 名詞 (Nouns)

| 英単語 | 日本語訳 | プログラミングにおける意味 | 使用例 | 使用例の日本語訳 |
|--------|----------|--------------------------|--------|------------------|
| variable | 変数、可変の | データを格納するための名前付きのメモリ領域。値が変更可能。 | Declare a variable to store the value | 値を格納する変数を宣言する |
| condition | 条件、状態 | プログラムの実行フローを制御するための真偽値の式。if文、while文などで使用。 | Check the condition before executing | 実行前に条件をチェックする |
| argument | 引数、議論 | 関数やメソッドを呼び出す際に渡す実際の値や変数。 | Provide arguments when calling the function | 関数を呼び出す際に引数を提供する |
| exception | 例外、異常 | プログラム実行中に発生する予期しないエラーや異常な状態。try-catch文で処理。 | Handle the exception properly | 例外を適切に処理する |
| syntax | 構文、文法 | プログラミング言語の文法規則。コードの書き方のルール。 | Check the syntax of your code | コードの構文をチェックする |
| statement | 文、声明 | プログラムの実行単位。1つの命令や処理を表す。 | Each statement ends with a semicolon | 各文はセミコロンで終わる |
| expression | 式、表現 | 値を計算するコードの断片。変数、演算子、関数呼び出しの組み合わせ。 | Evaluate the expression | 式を評価する |
| operator | 演算子、操作者 | 値に対して演算を行う記号。+（加算）、-（減算）、==（等価）など。 | Use the comparison operator | 比較演算子を使用する |
| operand | 被演算子、オペランド | 演算子によって操作される値や変数。 | The operands of the operation | 操作の被演算子 |
| declaration | 宣言、表明 | 変数や関数の存在をコンパイラに知らせること。型や名前を指定。 | Variable declaration | 変数の宣言 |
| definition | 定義、明確化 | 変数や関数の実装を記述すること。宣言に加えて実体を定義。 | Function definition | 関数の定義 |
| assignment | 代入、割り当て | 変数に値を格納すること。=演算子などで実行。 | Assignment operator | 代入演算子 |
| dependency | 依存関係、依存性 | プログラムが動作するために必要な他のライブラリやパッケージ。 | Manage project dependencies | プロジェクトの依存関係を管理する |
| data structure | データ構造、データの構造 | データを効率的に格納・操作するための構造。配列、リスト、ツリー、ハッシュテーブルなど。 | Choose the right data structure | 適切なデータ構造を選択する |
| vertex | 頂点、頂 | グラフのノード。頂点と同義。 | Find the vertex | 頂点を見つける |
| element | 要素、成分 | データ構造に格納される個々の値やオブジェクト。 | Access each element | 各要素にアクセスする |
| character | 文字、性格 | 1つの文字。文字列を構成する最小単位。 | Check each character | 各文字をチェックする |
| capacity | 容量、能力 | データ構造が格納できる最大要素数。動的配列などで使用。 | Set the capacity | 容量を設定する |
| inheritance | 継承、相続 | クラスが他のクラスの属性やメソッドを受け継ぐこと。コードの再利用と階層構造を実現。 | Use inheritance to extend classes | クラスを拡張するために継承を使用する |
| polymorphism | 多態性、多様性 | 同じインターフェースで異なる実装を扱えること。実行時に適切なメソッドが呼ばれる。 | Implement polymorphism | 多態性を実装する |
| encapsulation | カプセル化、封じ込め | データとメソッドを1つの単位にまとめ、内部実装を隠蔽すること。情報隠蔽の原則。 | Achieve encapsulation | カプセル化を実現する |
| abstraction | 抽象化、抽出 | 複雑な実装の詳細を隠し、本質的な機能だけを提供すること。抽象クラスやインターフェースで実現。 | Create an abstraction | 抽象化を作成する |
| attribute | 属性、特性 | オブジェクトやクラスの特徴を表すデータ。フィールドやプロパティと同義。 | Set the attribute | 属性を設定する |
| type constraint | 型制約 | ジェネリックの型パラメータに課す制約。特定のインターフェースの実装など。 | Add type constraints | 型制約を追加する |
| type inference | 型推論 | コンパイラが自動的に型を推論すること。明示的な型指定が不要になる。 | Use type inference | 型推論を使用する |
| type safety | 型安全性 | コンパイル時に型の整合性が保証されること。型エラーを事前に検出できる。 | Ensure type safety | 型安全性を確保する |
| side effect | 副作用、余波 | 関数の戻り値以外に外部の状態を変更すること。グローバル変数の変更、ファイルの書き込みなど。 | Avoid side effects | 副作用を避ける |
| partial application | 部分適用 | 関数の一部の引数に値を固定して、残りの引数を取る新しい関数を作ること。 | Use partial application | 部分適用を使用する |
| composition | 合成、構成 | 複数の関数を組み合わせて新しい関数を作ること。関数の連結。 | Function composition | 関数の合成 |
| recursion | 再帰、回帰 | 関数が自分自身を呼び出すこと。ループの代替手段。 | Use recursion | 再帰を使用する |
| tail recursion | 末尾再帰 | 再帰呼び出しが関数の最後にある再帰。スタックオーバーフローを防げる。 | Optimize with tail recursion | 末尾再帰で最適化する |
| predicate | 述語、断定 | 真偽値を返す関数。条件判定に使用。filterなどで使用。 | Define a predicate function | 述語関数を定義する |
| bandwidth | 帯域幅、通信容量 | ネットワークで転送できるデータ量。ビット/秒で表す。 | Check bandwidth usage | 帯域幅の使用量をチェックする |
| isolation | 分離、孤立 | トランザクションが他のトランザクションから独立して実行されること。ACID特性の1つ。 | Transaction isolation | トランザクション分離 |
| consistency | 一貫性、整合性 | データベースの整合性が保たれること。ACID特性の1つ。 | Maintain consistency | 一貫性を維持する |
| durability | 永続性、耐久性 | コミットされたトランザクションの結果が失われないこと。ACID特性の1つ。 | Ensure durability | 永続性を確保する |
| atomicity | 原子性 | トランザクションが全て実行されるか、全て実行されないかのいずれかであること。ACID特性の1つ。 | Guarantee atomicity | 原子性を保証する |
| relationship | リレーションシップ、関係 | テーブル間の関連性。1対1、1対多、多対多の関係。 | Define relationships | リレーションシップを定義する |
| normalization | 正規化、標準化 | データベースの設計手法。データの冗長性を削減し、整合性を保つ。第1正規形から第5正規形まで。 | Apply normalization | 正規化を適用する |
| authorization | 認可、権限付与 | 認証されたユーザーやシステムが特定のリソースにアクセスできる権限を付与すること。 | Check authorization | 認可をチェックする |
| encryption | 暗号化、符号化 | データを保護するための暗号化処理。AES、RSAなどのアルゴリズムを使用。 | Use encryption | 暗号化を使用する |
| decryption | 復号化、解読 | 暗号化されたデータを元に戻す処理。 | Perform decryption | 復号化を実行する |
| permission | 権限、許可 | リソースへのアクセスや操作を許可する権利。読み取り、書き込み、実行など。 | Check permissions | 権限をチェックする |
| access control | アクセス制御 | リソースへのアクセスを制限・管理する仕組み。認証と認可を組み合わせて実現。 | Implement access control | アクセス制御を実装する |
| vulnerability | 脆弱性、弱点 | システムのセキュリティ上の欠陥。攻撃の標的となる可能性がある。 | Fix vulnerabilities | 脆弱性を修正する |
| attack | 攻撃、襲撃 | システムやデータに害を与える行為。DDoS攻撃、SQLインジェクションなど。 | Prevent attacks | 攻撃を防ぐ |
| certificate | 証明書、証書 | 公開鍵と所有者情報を含む電子証明書。SSL/TLS通信で使用。 | Install SSL certificate | SSL証明書をインストールする |
| signature | 署名、サイン | データの真正性を証明するための電子署名。デジタル署名で使用。 | Verify the signature | 署名を検証する |
| threat | 脅威、恐れ | システムやデータに対する潜在的な危険。セキュリティ脅威の分析が必要。 | Identify threats | 脅威を特定する |
| intrusion | 侵入、侵害 | 不正なアクセスや侵入の試み。侵入検知システム（IDS）で検出。 | Detect intrusions | 侵入を検出する |
| persistent volume | 永続ボリューム | データが永続的に保存されるボリューム。Podが削除されてもデータは残る。 | Use persistent volumes | 永続ボリュームを使用する |
| utilization | 利用率、使用率 | リソースの使用割合。CPU利用率、メモリ利用率など。監視の指標。 | Monitor utilization | 利用率を監視する |
| monitoring | 監視、観察 | システムの状態やパフォーマンスを継続的に監視すること。 | Set up monitoring | 監視を設定する |
| log aggregation | ログ集約 | 複数のソースからのログを1箇所に集約すること。分析や検索を容易にする。 | Set up log aggregation | ログ集約を設定する |
| assignee | 担当者、割り当て先 | イシューやPRの担当者。作業を割り当てる。 | Set assignee | 担当者を設定する |
| discussion | 議論、討論 | コードや機能についての議論。GitHub Discussionsなど。 | Start discussion | 議論を開始する |
| package | パッケージ、包装 | 再利用可能なコードの単位。npmパッケージ、Pythonパッケージなど。 | Install package | パッケージをインストールする |
| integration test | 統合テスト | 複数のコンポーネントを組み合わせてテストすること。 | Write integration tests | 統合テストを書く |
| acceptance test | 受け入れテスト | 要件を満たしているかを確認するテスト。ユーザーストーリーに基づく。 | Run acceptance tests | 受け入れテストを実行する |
| regression test | 回帰テスト | 既存の機能が壊れていないかを確認するテスト。リグレッションテスト。 | Run regression tests | 回帰テストを実行する |
| load test | 負荷テスト | 通常の負荷でシステムが動作するかをテストすること。 | Run load tests | 負荷テストを実行する |
| race condition | 競合状態 | 複数のスレッドが共有リソースにアクセスして予期しない結果になる状態。 | Fix race condition | 競合状態を修正する |
| undefined behavior | 未定義動作 | 言語仕様で動作が定義されていない状態。予測不可能な結果。 | Avoid undefined behavior | 未定義動作を避ける |
| assertion failure | アサーション失敗 | アサーションが失敗すること。期待と実際が異なる。 | Fix assertion failure | アサーション失敗を修正する |
| flaky test | 不安定なテスト | 時々失敗するテスト。再現性が低い。環境やタイミングに依存。 | Fix flaky test | 不安定なテストを修正する |
| steps to reproduce | 再現手順 | バグを再現するための手順。バグレポートに記載。 | Provide steps to reproduce | 再現手順を提供する |
| workaround | 回避策、代替手段 | 問題を一時的に回避する方法。根本的な解決ではない。 | Use workaround | 回避策を使用する |
| regression | 回帰、後退 | 以前動作していた機能が壊れること。リグレッション。 | Fix regression | 回帰を修正する |
| root cause | 根本原因 | 問題の根本的な原因。表面的な症状ではない。 | Find root cause | 根本原因を見つける |
| continuous integration | 継続的インテグレーション | コードの変更を頻繁に統合してテストすること。早期に問題を発見。 | Use CI | CIを使用する |
| continuous delivery | 継続的デリバリー | いつでも本番環境にデプロイできる状態を保つこと。手動でデプロイ。 | Use CD | CDを使用する |
| continuous deployment | 継続的デプロイメント | テストが通れば自動的に本番環境にデプロイすること。完全自動化。 | Enable CD | CDを有効にする |
| notification | 通知、知らせ | CI/CDの結果を通知すること。メール、Slackなど。 | Send notification | 通知を送信する |
| force-unlock | 強制アンロック | ロックを強制的に解除すること。デッドロックの解決。 | Force unlock | 強制アンロックする |
| reference | 参照、言及 | オブジェクトへの参照。参照型の変数。 | Object reference | オブジェクト参照 |
| situation | 状況、事態 | 特定の状況。エラーハンドリングなどで使用。 | Handle situation | 状況を処理する |
| structure | 構造、建物 | データ構造。配列、リスト、ツリーなど。 | Data structure | データ構造 |
| contract | 契約、契約書 | サービス間の契約。APIの仕様など。 | Service contract | サービス契約 |
| specification | 仕様、明細 | システムやAPIの仕様。動作の定義。 | Read specification | 仕様を読む |
| requirement | 要件、要求 | システムの要件。機能要件、非機能要件。 | Meet requirement | 要件を満たす |
| feature | 機能、特徴 | アプリケーションの機能。新機能の追加。 | Add feature | 機能を追加する |
| capability | 能力、才能 | システムの能力。機能と同義。 | Add capability | 能力を追加する |
| functionality | 機能性、機能 | システムの機能性。動作する機能。 | Test functionality | 機能性をテストする |
| behavior | 動作、振る舞い | オブジェクトやシステムの動作。期待される動作。 | Observe behavior | 動作を観察する |
| operation | 操作、運転 | データベース操作やAPI操作など。 | Execute operation | 操作を実行する |
| process | プロセス、過程 | 実行中のプログラム。OSが管理する実行単位。 | Start process | プロセスを開始する |
| outcome | 結果、成果 | 操作の結果。resultと同義。 | Check outcome | 結果を確認する |
| record | レコード、記録 | データベースのレコード。1行のデータ。 | Update record | レコードを更新する |
| association | 関連、連想 | オブジェクト間の関連。関連付け。 | Create association | 関連を作成する |
| translation | 翻訳、変換 | データの変換。座標変換など。 | Perform translation | 変換を実行する |

| 英単語 | 日本語訳 | プログラミングにおける意味 | 使用例 | 使用例の日本語訳 |
|--------|----------|--------------------------|--------|------------------|
| execute | 実行する、遂行する | プログラムやコマンドを実際に動作させること。 | Execute the program | プログラムを実行する |
| implement | 実装する、実行する | インターフェースや仕様に基づいて実際のコードを書くこと。 | Implement the interface | インターフェースを実装する |
| define | 定義する、明確にする | 変数、関数、クラスなどを定義すること。名前と実装を結びつける。 | Define the function | 関数を定義する |
| declare | 宣言する、表明する | 変数や関数の存在を宣言すること。型や名前を指定。 | Declare the variable | 変数を宣言する |
| assign | 代入する、割り当てる | 変数に値を格納すること。=演算子などで実行。 | Assign a value to the variable | 変数に値を代入する |
| invoke | 呼び出す、起動する | 関数やメソッドを実行すること。引数を渡して呼び出す。 | Invoke the method | メソッドを呼び出す |
| iterate | 反復する、繰り返す | コレクションの要素を順番に処理すること。ループで実行。 | Iterate through the collection | コレクションを反復処理する |
| traverse | 走査する、横断する | データ構造（特にツリーやグラフ）の全ての要素を訪問すること。 | Traverse the tree | ツリーを走査する |
| reduce | リデュースする、減らす | コレクションを1つの値に集約すること。合計、最大値などを計算。 | Reduce the collection | コレクションをリデュースする |
| transform | 変換する、変形する | データの形式や構造を変更すること。 | Transform the data | データを変換する |

| decode | デコードする、復号化する | エンコードされたデータを元の形式に戻すこと。 | Decode the data | データをデコードする |
| overload | オーバーロードする、多重定義する | 同じ名前で異なる引数のメソッドを複数定義すること。引数の型や数が異なる。 | Overload the function | 関数をオーバーロードする |
| extend | 拡張する、広げる | クラスを継承すること。親クラスの機能を引き継ぎ、追加の機能を実装。 | Extend the base class | 基底クラスを拡張する |
| inherit | 継承する、受け継ぐ | 親クラスから属性やメソッドを受け継ぐこと。 | Inherit from the parent class | 親クラスから継承する |
| derive | 派生する、導出する | 基底クラスから新しいクラスを作成すること。継承と同義。 | Derive a new class | 新しいクラスを派生する |
| cast | キャストする、型変換する | ある型を別の型に変換すること。明示的な型変換。 | Cast to the target type | ターゲット型にキャストする |
| fold | 畳み込む、折りたたむ | コレクションを1つの値に集約すること。reduceと同義。 | Fold the collection | コレクションを畳み込む |
